## 1. 全局说明

### 1.1 统一响应格式

```json
{
  "code": 0,
  "message": "ok",
  "data": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 业务错误码，`0` 代表成功 |
| `message` | string | 响应描述 |
| `data` | any | 业务数据，无数据返回 `null` |

分页接口的 `data` 结构：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "list": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 20
  }
}
```

### 1.2 请求 Header

所有业务接口（`/health` 除外）必须携带：

| Header | 必填 | 说明 |
|--------|------|------|
| `Authorization` | ✅ | 身份凭证 Token（JWT） |
| `X-Trace-Id` | ✅ | 全链路追踪 ID，ULID 格式 |
| `X-Request-Source` | ✅ | 请求来源：`client` / `scheduler` / `third_party` |
| `X-Timestamp` | ✅ | Unix 秒级时间戳（防重放） |
| `X-Signature` | ✅ | HMAC-SHA256 签名（防重放） |

> 写操作（POST/PUT/PATCH）建议额外传递 `Idempotency-Key` 实现幂等。

### 1.3 ID 生成规则

全局使用 ULID 算法，数据库统一 `varchar(64)`：

| 前缀 | 业务实体 | 示例 |
|------|---------|------|
| `ep_` | 医疗疗程 | `ep_01JZYQ4M8PABCDEFGHJKLMNPQR` |
| `task_` | 护理任务 | `task_01JZYQ5BXXXXXYZ...` |
| `sum_` | 医疗摘要 | `sum_01JZYQ7D...` |
| `auth_` | 医疗授权 | `auth_01JZYQ8E...` |
| `opp_` | 代遛需求 | `opp_01JZYQ9F...` |
| `plan_` | 代遛方案 | `plan_01JZYQ0G...` |
| `order_` | 代遛订单 | `order_01JZYQ1H...` |
| `report_` | 服务报告 | `report_01JZYQ3J...` |

### 1.4 时间格式

- 统一 **UTC+8（东八区）**
- 格式：`yyyy-MM-ddTHH:mm:ss+08:00`
- 示例：`2026-06-16T10:30:00+08:00`

### 1.5 分页参数

列表类接口统一使用查询参数：

| 参数 | 类型 | 默认 | 最大 | 说明 |
|------|------|------|------|------|
| `pageNum` | int | 1 | - | 页码，从 1 开始 |
| `pageSize` | int | 20 | 50 | 每页条数 |

### 1.6 错误码速查

| 码值 | 含义 | 场景 |
|------|------|------|
| `0` | 成功 | - |
| `1001` | 未登录/Token 无效 | Header 校验 |
| `1002` | 无权限 | 鉴权 |
| `1003` | 缺少必要 Header | Header 校验 |
| `2001` | 医疗疗程不存在 | 医疗模块查询 |
| `2002` | 非法状态转换 | 医疗模块状态机 |
| `2003` | 无医疗数据授权 | 推送诊所 |
| `3001` | 代遛需求不存在 | 代遛模块查询 |
| `3002` | 非法状态转换 | 代遛模块状态机 |
| `3003` | 禁止自动下单 | 代遛风控 |
| `4001` | 支付失败 | 支付回调 |

### 2.1 获取宠物当前医疗疗程

查询指定宠物当前进行中的医疗疗程。

```
GET /api/v1/pets/{pet_id}/medical-episodes/current
```

**路径参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pet_id` | string | ✅ | 宠物 ID |

**Curl 示例**

```bash
curl -X GET 'http://localhost:8080/api/v1/pets/pet_xxx/medical-episodes/current' \
  -H 'Authorization: Bearer <token>' \
  -H 'X-Trace-Id: trace_01JZYQ...' \
  -H 'X-Request-Source: client' \
  -H 'X-Timestamp: 1718520000' \
  -H 'X-Signature: <hmac_hex>'
```

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "ep_01JZYQ4M8PABCDEFGHJKLMNPQR",
    "family_id": "fam_xxx",
```

### 2.3 生成医嘱任务草稿

基于医嘱生成护理任务草稿。仅 `EpisodeCreated` 状态的疗程允许操作，成功后状态推进至 `InstructionDrafted`。

```
POST /api/v1/medical-episodes/{episode_id}/care-tasks/draft
```

**路径参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `episode_id` | string | ✅ | 疗程 ID |

**请求体**

```json
{
  "tasks": [
    {
      "task_type": "medication",
      "title": "喂药：阿莫西林 250mg",
      "instruction": "每日两次，每次一片，饭后服用",
      "due_at": "2026-06-17T08:00:00+08:00",
      "risk_level": "P1",
      "lock_fields": ["medication_name", "dosage"]
    },
    {
      "task_type": "observation",
      "title": "观察精神状态",
      "instruction": "记录每日精神状态变化",
      "due_at": "2026-06-17T20:00:00+08:00",
      "risk_level": "P2"
    }
  ]
}
```

**请求字段说明**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `tasks` | array | ✅ | 任务列表，至少 1 项 |
| `tasks[].task_type` | string | ✅ | 任务类型：`medication` / `care_measure` / `observation` / `exercise` |
| `tasks[].title` | string | ✅ | 任务标题 |
| `tasks[].instruction` | string | - | 执行说明 |
| `tasks[].due_at` | string | - | 截止时间（UTC+8） |
| `tasks[].risk_level` | string | - | 风险等级：`P0` / `P1` / `P2` |
| `tasks[].lock_fields` | string[] | - | 需要锁定的字段（AI 不可修改） |

**Curl 示例**

```bash
curl -X POST 'http://localhost:8080/api/v1/medical-episodes/ep_xxx/care-tasks/draft' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -H 'X-Trace-Id: trace_01JZYQ...' \
  -H 'X-Request-Source: client' \
  -H 'X-Timestamp: 1718520000' \
  -H 'X-Signature: <hmac_hex>' \
  -H 'Idempotency-Key: draft_ep_xxx_20260616_001' \
  -d '{
    "tasks": [
      {
        "task_type": "medication",
        "title": "喂药：阿莫西林 250mg",
        "instruction": "每日两次，每次一片，饭后服用",
        "due_at": "2026-06-17T08:00:00+08:00",
        "risk_level": "P1"
      }
    ]
  }'
```

**响应 `201 Created`**

```json
{
  "code": 0,
  "message": "任务草稿生成成功",
  "data": {
    "episode_id": "ep_01JZYQ...",
    "tasks": [
      {
        "id": "task_01JZYQ...",
        "episode_id": "ep_01JZYQ...",
        "task_type": "medication",
        "title": "喂药：阿莫西林 250mg",
        "instruction": "每日两次，每次一片，饭后服用",
        "status": "Draft",
        "risk_level": "P1",
        "locked_fields_json": "[\"medication_name\",\"dosage\"]",
        "due_at": "2026-06-17T08:00:00+08:00",
        "created_by": "agent",
        "created_at": "2026-06-16T10:30:00+08:00",
        "updated_at": "2026-06-16T10:30:00+08:00"
      }
    ]
  }
}
```

**错误码**

| HTTP 状态码 | code | message |
|-------------|------|---------|
| 400 | 2001 | episode_id 不能为空 |
| 400 | 2002 | 请求参数无效 |
| 400 | 2002 | 当前状态不允许生成任务草稿（仅 `EpisodeCreated` 允许） |
| 404 | 2001 | 疗程不存在 |
| 500 | 2001 | 查询/保存失败 |

### 2.4 标记任务完成

护理任务完成后标记为已完成。仅 `Draft` / `Active` 状态的任务允许操作。重复请求（相同 `idempotency_key`）幂等返回首次结果。

```
POST /api/v1/care-tasks/{task_id}/complete
```

**路径参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_id` | string | ✅ | 任务 ID |

**请求体**

```json
{
  "idempotency_key": "complete_task_xxx_20260616_001",
  "note": "已按时完成喂药"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `idempotency_key` | string | ✅ | 幂等键，重复请求返回首次结果 |
| `note` | string | - | 完成备注 |

**Curl 示例**

```bash
curl -X POST 'http://localhost:8080/api/v1/care-tasks/task_xxx/complete' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -H 'X-Trace-Id: trace_01JZYQ...' \
  -H 'X-Request-Source: client' \
  -H 'X-Timestamp: 1718520000' \
  -H 'X-Signature: <hmac_hex>' \
  -d '{
    "idempotency_key": "complete_task_xxx_20260616_001",
```

### 2.6 摘要推送至诊所

将已生成的医疗摘要推送到指定诊所。需要先完成数据授权状态机（必须先创建授权，且授权有效）。

```
POST /api/v1/medical-summaries/{summary_id}/send-to-clinic
```

**路径参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `summary_id` | string | ✅ | 摘要 ID |

**请求体**

```json
{
  "clinic_id": "clinic_xxx"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `clinic_id` | string | ✅ | 目标诊所 ID |

**前置条件**

1. 关联疗程状态必须为 `AwaitingAuthorization`
2. 必须有 `status = "active"` 的有效授权记录

**完整业务流程（5 步）**

```
① 创建疗程（外部）→ ② 生成任务草稿 → ③ 标记任务完成
→ ④ 生成摘要 → ⑤ 创建授权 → ⑥ 推送诊所
```

**Curl 示例**

```bash
curl -X POST 'http://localhost:8080/api/v1/medical-summaries/sum_xxx/send-to-clinic' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -H 'X-Trace-Id: trace_01JZYQ...' \
  -H 'X-Request-Source: client' \
  -H 'X-Timestamp: 1718520000' \
  -H 'X-Signature: <hmac_hex>' \
  -d '{
    "clinic_id": "clinic_xxx"
  }'
```

**响应 `200 OK`**

```json
{
  "code": 0,
  "message": "摘要已推送诊所",
  "data": {
    "summary_id": "sum_01JZYQ...",
    "clinic_id": "clinic_xxx",
    "episode_id": "ep_01JZYQ..."
  }
}
```

**错误码**

| HTTP 状态码 | code | message |
|-------------|------|---------|
| 400 | 2001 | summary_id 不能为空 |
| 400 | 2002 | 请求参数无效 |
| 400 | 2003 | 当前状态不允许推送诊所，需要先完成授权 |
| 403 | 2003 | 无有效授权，无法推送诊所 |
| 404 | 2001 | 摘要/疗程不存在 |
| 500 | 2001 | 查询/更新失败 |
