# 03 帕奇Agent AI编码规范

## 通用强制规则（全局生效）

1. 本文档为最终标准，Ralph-Claude-Code 禁止擅自新增功能、修改流程、删减约束；

2. 所有代码、配置、脚本禁止出现明文密钥、IP、端口、账号等敏感信息；

3. 所有时间字段统一使用 **UTC+8（东八区）** 时间格式：`yyyy-MM-ddTHH:mm:ss+08:00`；

4. 日志输出统一结构化JSON格式，禁止纯文本日志。

## 文档说明

本文为 `frankbria/ralph-claude-code` 专用编码规范，约束所有AI生成代码行为。

## 一、整体技术栈（禁止修改）

### 1.1 主服务（Go）

- 编程语言：Go 1.24

- Web框架：Gin

- ORM：GORM

- Agent编排：LangGraph + 自研状态机

- 禁止使用：LangChain AgentExecutor、AutoGPT、CrewAI

### 1.2 AI服务（Python）

- 承载能力：LLM推理、RAG、Prompt、AI评测、离线脚本

- 禁止用于：线上HTTP接口、交易、医疗核心流程

## 二、双语言分工规则

### 2.1 仅使用Go开发（线上核心）

API网关、认证、权限、状态机、Saga事务、数据库、Redis、Kafka、支付、诊所/服务商对接、生命流、审计、配置中心、功能开关。

### 2.2 仅使用Python开发（AI/离线）

LLM调用、RAG向量化/检索、Prompt管理、AI自动化评测、数据清洗、离线运维脚本。

### 2.3 强制禁止

1. 同一文件混编两种语言；

2. Go直接调用Python本地函数；

3. Python编写线上交易/医疗接口；

4. 两种服务部署在同一进程/Pod。

## 四、命名规范

1. ID规则：全局ULID + 固定前缀，参考接口契约；

2. Go代码：变量/方法/结构体=小驼峰，常量/枚举=大驼峰；

3. 文件/目录：全小写，下划线分隔；

4. 数据库/JSON字段：统一小驼峰。

## 五、状态机代码规范

### 5.1 实现模式

无状态状态机，**禁止大量手写switch-case**，流转配置化。

## 六、通用代码约束

1. 架构模式：DDD领域分层 + Repository + Service层；

2. 禁止业务代码直连数据库，必须通过Repository；

3. 所有写操作强制添加幂等键；

4. 医疗、支付高危代码添加 `@Risk(P0)` 注解；

5. 所有工具调用必须经过工具网关；

6. 高危操作强制写入审计日志。

## 七、配置管理规范

1. 所有超时、限流、地址、密钥、阈值统一托管 **Nacos**；

2. 灰度开关统一使用 **Unleash**；

3. 代码**禁止硬编码**任何配置、IP、域名。

## 八、Prompt版本管理

1. 存放目录：`ai-service/prompt`

2. 命名规则：`prompt_v1.0.yaml`、`prompt_v1.1.yaml`

3. 规则：支持动态加载、版本回滚，历史版本至少保留3份。

## 九、AI评测规范

### 9.1 评测集规模

1. 医疗评测集：1000条（注入、工具滥用、隐私、问诊、改剂量，各200条）

2. 代遛评测集：1000条（自动下单、地址泄露、重复扣款等，各200条）

### 9.2 验收标准

P0违规场景拦截率 100%

### 9.3 单元测试细则

1. 核心模块单元测试覆盖率必须100%；

2. 用例覆盖正常、边界、异常、越权场景；

3. AI生成代码**同步生成单元测试**。

## 十、熔断&重试规范

1. 第三方接口：3s超时，连续10次失败熔断5分钟；

2. 普通接口：重试2次；

3. LLM：重试1次；

4. 消息队列：按队列配置执行重试。

## 十一、数据备份规范

1. PostgreSQL：每日全量 + 每小时增量，保留30天；

2. TimescaleDB：分区归档，保留90天；

3. 备份文件异地存储，定期恢复演练。

## 十二、版本兼容规则

1. 接口、数据表向前兼容，新增字段允许为空/设默认值，禁止删改原有字段；

2. Prompt、LLM输出Schema迭代保持旧版本兼容。

## 十三、安全编码强制规则

1. SQL防护：GORM参数化查询，**禁止拼接SQL**，防注入；

2. XSS防护：后端对回显数据做特殊字符转义；

3. 敏感数据：手机号/地址/医疗隐私**禁止明文打印日志**；

4. 权限：所有接口入口优先执行权限校验；

5. 入参：强制做长度、格式、范围校验。

## 十四、日志编码规范

1. Go日志库：zap；Python日志库：logging；

2. 日志级别：DEBUG/INFO/WARN/ERROR；

3. ERROR日志必须携带 `trace_id`；

4. 禁止打印SQL、Token、密钥、隐私数据；

5. 框架日志与业务日志分离。

## 十五、Ralph执行强制规则

1. 严格遵循目录、技术栈、语法；

2. 不新增目录、接口、枚举；

3. 代码/SQL/复杂逻辑必须加注释；

4. 编码完成后全量回归校验。
