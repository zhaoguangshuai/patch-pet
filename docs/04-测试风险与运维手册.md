## 文档说明

本文包含测试体系、风险评估、合规、运维、上线、应急全配套内容。

## 一、全维度测试体系

### 1.1 分层测试

1. 单元测试：状态机、权限、工具、拦截，核心覆盖率100%；

2. 接口自动化：正向、异常、越权、幂等场景；

3. 集成测试：端到端、双Agent冲突场景；

4. UI原型对齐：交互、文案、状态匹配原型；

5. 性能压测：接口99分位延迟≤500ms，并发报错率=0。

### 1.2 AI专项评测

参考编码规范，P0违规拦截率必须100%。

### 1.3 上线验收指标

单元覆盖率≥90%、自动化用例通过率100%、红线拦截100%、原型匹配100%。

## 二、全维度风险评估矩阵

### 2.1 P0 致命风险

|风险描述|应对预案|告警时效|
|---|---|---|
|LLM输出诊断/改剂量|双层网关拦截，违规灰度下线|5分钟响应|
|支付幂等失效、重复扣款|分布式锁+每日自动对账退款|5分钟响应|
|医疗/轨迹数据泄露|数据脱敏+访问拦截|5分钟响应|
|无授权外发医疗数据|状态机强制拦截|5分钟响应|

### 2.2 P1 高风险

服务商接口宕机、定时任务丢失、路由错乱、地图接口失效；配套熔断、补提醒、告警方案。

### 2.3 P2 一般风险

LLM超时、服务商排序异常；依托监控+迭代优化。

### 2.4 应急流程

告警触发 → 分级响应 → 熔断阻断风险 → 排查根因 → 复盘优化。

### 2.5 P0故障应急SOP

1. 告警触发，运维5分钟内响应并拉起应急群；

2. 第一时间熔断故障模块/关闭功能开关，阻断影响；

3. 结合Trace、日志、审计日志定位问题；

4. 优先版本回滚（5分钟内完成）；

5. 故障复盘，输出优化报告。

## 三、数据生命周期&恢复规则

### 3.1 数据留存规则

|数据类型|保存时长|存储要求|
|---|---|---|
|医疗记录|5年|加密存储|
|授权/支付记录|7年|防篡改存储|
|轨迹定位数据|180天|脱敏，自动删除|
|审计日志|1年|只读归档|

### 3.2 用户注销规则

7个工作日内完成：脱敏 → 归档 → 可删除数据销毁。

### 3.3 数据恢复SOP

1. 误删数据：使用定时备份恢复（单表/单记录）；

2. 恢复权限：仅运维+数据库管理员可操作；

3. 恢复后全量校验数据完整性。

## 四、伦理&用户体验约束

1. 医疗状态下关闭所有商业弹窗；

2. 禁止情绪操纵、制造焦虑、诱导消费；

3. 医疗输出强制附加免责声明；

4. 服务商排序透明，禁止付费置顶。

## 五、第三方对接规范

1. 统一封装SDK，禁止直连；

2. 超时3s，失败重试2次；

3. 连续10次失败触发5分钟熔断；

4. 密钥/地址存放配置中心，禁止硬编码；

5. 第三方不可用时，前端展示统一兜底文案。

## 六、研发里程碑与分工

### 6.1 里程碑

1. M0：工程Demo（2周）

2. M1：正式MVP（4-6周）

3. M2：生产灰度（4周）

### 6.2 角色分工

产品、后端、算法、前端、测试、运维、安全。

### 6.3 灰度发布规则

1. 灰度维度：用户分组、地域、Unleash功能开关；

2. 放量比例：5% → 20% → 50% → 100%；

3. 观察周期：至少2小时，无P0/P1告警再放量；

4. 回滚：一键关闭开关+版本回滚。

## 八、上线&回滚检查清单

### 8.1 医疗模块

边界文案、字段锁定、授权校验、数据完整性、异常兜底。

### 8.2 代遛模块

服务商拦截、支付幂等、预约规则、数据脱敏、异常提醒。

### 8.3 工程运维

状态机、告警、功能开关、第三方连通、回滚脚本、配置校验。

### 8.4 版本回滚方案

1. 优先级：关闭功能开关 > 配置回滚 > 镜像回滚；

2. K8s保留上一稳定镜像，支持一键切换；

3. 回滚后执行核心流程回归测试。

## 九、日常运维巡检

1. 每日巡检：服务状态、接口错误率、磁盘、内存、CPU、消息队列堆积；

2. 每周巡检：备份有效性、权限列表、服务商状态；

3. 每月巡检：合规审计、数据留存、配置核对。
